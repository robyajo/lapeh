Catatan Penting: Saat ini, Dashboard menggunakan Mock Data (data simulasi) karena Anda belum memiliki server database backend yang aktif untuk menampung data telemetry ini. Kode di CLI ( bin/index.js ) sudah siap mengirim data ke https://lapeh-doc.vercel.app/api/telemetry .

Langkah selanjutnya untuk membuat ini "Live":

1. Buat API Route di Vercel ( api/telemetry.js ) atau server backend terpisah.

2. Hubungkan ke database (seperti Supabase/PostgreSQL).

3. Update URL di bin/index.js dan AdminDashboard.vue ke API asli Anda.

Untuk sekarang, Anda bisa membuka /admin di website lokal atau setelah deploy untuk melihat bentuk visual dashboard tersebut.

E. Lakukan testing (Folder pengujian local dan npm) dan pengujian framework selesai itu lanjutkan yang dibawah
1. Perbaharui CHANGELOG.md dengan perbaikan terbaru
2. Naikan versi di package.json (bumppp version)
3. jalankan npm publish (atau perintah final)
4. Buat postingan blog baru di website tentang rilis ini
5. commit ke github
6. perbahrui juga doc id dan en

D. Setelah semua di lakukan lakukan pengujian sebagai user ( lakukan semua itu dalam folder pengjujian) pastikan semua berjalan lancar

 2. npx lapeh@latest nama-project-anda --full
 3. npx lapeh@latest nama-project-anda --default
 4. npx lapeh@latest nama-project-anda --y
